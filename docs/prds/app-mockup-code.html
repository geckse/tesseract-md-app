# Markdown Database Editor View

---
type: specification status: in-progress tags: [ui, design, v1] updated: 2023-10-27
---

This is a **soft-render view** of the document. The goal is to provide a distraction-free environment that still hints at the underlying Markdown structure.

## Core Components

  • - The `Sidebar` organizes the [[Knowledge Base]] hierarchy.
  • - The `Editor` dims syntax characters to focus on content.
  • - Frontmatter properties drive the database queries.

## Design Principles

>

The interface should recede. Functionality should emerge only when needed.

We use a textured, neutral dark palette to reduce eye strain during long writing sessions. The primary accent color is strictly used for interactive elements and active states.

const palette = {

background: '#0f0f10',
